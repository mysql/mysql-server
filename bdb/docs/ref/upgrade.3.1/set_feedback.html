Release 3.1: DBENV->set_feedback, DB->set_feedback

Starting with the 3.1 release of Berkeley DB, the DBENV->set_feedback and DB->set_feedback functions may return an error value, that is, they are no longer declared as returning no value, instead they return an int or throw an exception as appropriate when an error occurs.

If your application calls these functions, you may want to check for a possible error on return.
